# Claude Code — Repository Learning Guide

Overview of knowledge extracted from [anthropics/claude-code](https://github.com/anthropics/claude-code).

This learning guide is designed for professionals who want to deeply understand the Claude Code ecosystem — its architecture, how it evolves, and how to contribute effectively. Whether you're exploring the plugin system, studying the CI/CD automation, or preparing to submit your first PR, this guide walks you through everything from the ground up.

## Subtopics

1. [Repo Goals & Architecture](repo-goals.md) — Purpose, design philosophy, plugin system, key technologies, and contribution model
2. [Claude Code as a User](cli-features.md) — CLI commands, flags, interactive features, sessions, permissions, and workflows
3. [Plugin Development Deep Dive](plugin-development.md) — Building custom plugins: commands, agents, skills, hooks, and MCP servers
4. [Repository Automation & Scripts](repo-automation.md) — Issue lifecycle, sweep scripts, gh.sh wrapper, GitHub Actions
5. [Contributing to Claude Code](contributing.md) — How to set up your environment, find good first issues, and submit PRs
6. [PR Review Insights](pr-reviews.md) — Code review patterns, key contributors, quality standards, and recurring themes
7. [The CHANGELOG as a Product Roadmap](changelog-roadmap.md) — Product evolution, release cadence, feature investment areas
8. [Recent Changes](recent-changes.md) — What's been happening in the last 2 weeks: new features, bug fixes, and release patterns

## Recommended Learning Path

1. **Start with Repo Goals** to understand the big picture — what Claude Code is, how it's structured, and where you fit in
2. **Learn to use Claude Code** before contributing — the CLI features guide grounds everything in practical experience
3. **Dive into Plugin Development** to understand the extension system — the primary contribution surface
4. **Study Repository Automation** to understand the other major contribution surface beyond plugins
5. **Follow Contributing to Claude Code** for the contribution process now that you understand both surfaces
6. **Read PR Review Insights** to learn the team's quality bar before submitting your own work
7. **Analyze the CHANGELOG** for a strategic view of where the product is heading
8. **Finish with Recent Changes** to see current momentum in context of everything you've learned
